# Frontend: план работы и как продолжить

Этот файл нужен, чтобы frontend можно было спокойно продолжить даже без Codex. Здесь описано, что уже заложено, как работать параллельно с backend и какие шаги делать дальше.

---

## 1. Ветка и зона ответственности

Frontend-работа ведется в отдельной ветке:

```bash
features/frontend-mvp
```

Твоя основная зона:

```text
frontend/
```

Чтобы не мешать другим:

- не трогай `backend/` без договоренности с backend-разработчиком;
- корневой `README.md` меняй только если нужно обновить общую документацию;
- API-контракт согласовывай заранее, а не через случайные изменения в коде;
- если меняешь общий файл вроде `.gitignore`, напиши команде.

---

## 2. Что сейчас создается

Frontend сделан максимально просто: обычные **HTML + CSS + JavaScript**.

React, Vite и npm сейчас не нужны. Это специально, чтобы можно было разобраться в проекте без знания frontend-фреймворков.

Для интерактивного графика подключен **Chart.js** через CDN. Если CDN не загрузится, в `app.js` есть простой fallback-график на обычном canvas.

Структура:

```text
frontend/
  README.md
  index.html
  styles.css
  app.js
  mockData.js
```

Главная идея: сначала frontend работает на демо-данных, поэтому backend не блокирует разработку интерфейса.

## 3. API-контракт с backend

Ожидаемый endpoint:

```text
GET http://localhost:8000/api/analyze/{ticker}
```

Пример:

```text
GET http://localhost:8000/api/analyze/AAPL
```

Ожидаемый JSON:

```json
{
  "ticker": "AAPL",
  "companyName": "Apple Inc.",
  "price": 190.12,
  "currency": "USD",
  "dailyChangePercent": 1.24,
  "priceHistory": [
    { "date": "20.05", "price": 184.1 },
    { "date": "21.05", "price": 185.4 },
    { "date": "22.05", "price": 183.9 }
  ],
  "companyDescription": "Краткое описание компании.",
  "news": [
    {
      "title": "Заголовок новости",
      "publishedAt": "2026-05-29",
      "summary": "Краткое содержание новости.",
      "url": "https://example.com",
      "impact": "neutral",
      "impactReason": "Нет сильного положительного или отрицательного сигнала."
    }
  ],
  "aiAnalysis": "Краткий итоговый аналитический отчет."
}
```

Если backend меняет названия полей, frontend тоже придется обновить.

## 4. Как запустить frontend

### Вариант 1. Самый простой

Открой файл в браузере:

```text
frontend/index.html
```

Демо-режим будет работать сразу.

### Вариант 2. Через локальный сервер

Этот вариант лучше, когда нужно подключать реальный backend.

Можно запустить сервер из корня проекта:

```bash
python -m http.server 5173
```

И открыть:

```text
http://localhost:5173
```

Корневой `index.html` сам перенаправит на frontend.

Либо можно запустить сервер прямо из папки frontend:

```bash
cd frontend
python -m http.server 5173
```

Тогда открывай:

```text
http://localhost:5173
```

Если при запуске сервера ты видишь список папок `frontend`, `backend`, `README.md`, значит сервер запущен из корня проекта и открылся режим просмотра папки. Открой адрес:

```text
http://localhost:5173/frontend/index.html
```

## 5. Демо-режим

По умолчанию frontend работает в демо-режиме.

Это значит:

- backend пока не нужен;
- работают только тикеры `AAPL`, `MSFT`, `TSLA`;
- для неизвестного тикера появляется ошибка;
- можно спокойно верстать интерфейс и состояния.

Настройка находится в файле:

```text
frontend/app.js
```

Сейчас там стоит:

```javascript
const USE_MOCK_API = true;
```

Когда backend будет готов, поменяй на:

```javascript
const USE_MOCK_API = false;
```

И проверь адрес backend:

```javascript
const API_BASE_URL = "http://localhost:8000";
```

После изменения обнови страницу в браузере.

### `index.html`

Это структура страницы.

Там находятся:

- заголовок;

- блок пустого состояния;
- карточка результата;
- график изменения цены;
- блок новостей;
- подключение `styles.css`, `mockData.js`, Chart.js и `app.js`.

Это внешний вид.

Там находятся:

- цвета;
- отступы;
- сетка блоков;
- адаптивность;
- стили ошибок, кнопок, карточек.

### `mockData.js`

Это тестовые данные.

Пока backend не готов, приложение берет данные отсюда.

### `app.js`

Это логика.

Там находятся:

- обработка формы;
- проверка тикера;
- mock-запрос;
- реальный `fetch` к backend;
- отрисовка интерактивного графика цены через Chart.js;
- вывод данных на страницу.

## 7. Твой план работы

### Шаг 1. Разобраться с файлами

Открой файлы в таком порядке:

1. `index.html`
2. `styles.css`
3. `mockData.js`
4. `app.js`

Не пытайся понять все сразу. Достаточно понять, какой файл за что отвечает.

### Шаг 2. Проверить запуск

Задача:

- открыть `index.html` в браузере;
- нажать кнопку `Анализ`;
- убедиться, что открывается главная страница.

Готово, если:

- поле ввода тикера видно;
- можно ввести `AAPL`;
- появляется карточка результата.

### Шаг 3. Довести интерфейс

Задача:

- проверить внешний вид карточки акции;
- проверить график изменения цены;
- проверить блок описания компании;
- проверить список новостей;
- проверить блок AI-анализа;
- улучшить отступы, размеры, адаптивность.

Готово, если:

- интерфейс понятно читается на ноутбуке;
- интерфейс не ломается на узком экране;
- ошибки и загрузка выглядят нормально.

### Шаг 4. Согласовать backend

Задача:

- передать backend-разработчику API-контракт из раздела 3;
- уточнить точный URL endpoint;
- уточнить, будут ли поля называться именно так.

Готово, если:

- backend возвращает JSON в ожидаемом формате;
- frontend-разработчик и backend-разработчик не спорят с кодом в руках.

### Шаг 5. Подключить реальный backend

Задача:

- в `app.js` поставить `USE_MOCK_API = false`;
- проверить `API_BASE_URL`;
- проверить запросы на `AAPL`, `MSFT`, `TSLA`.

Готово, если:

- реальные данные отображаются вместо демо-данных;
- ошибка backend показывается пользователю понятным текстом.

### Шаг 6. Финальная проверка

Проверить сценарии:

- пустой ввод;
- `AAPL`;
- `MSFT`;
- `TSLA`;
- несуществующий тикер;
- выключенный backend;
- длинный текст новости;
- отсутствие новостей.

Готово, если:

- приложение не падает;
- пользователь понимает, что происходит;
- результат можно показать на защите.

## 8. Что делать, если Codex больше недоступен

1. Работай только в ветке `features/frontend-mvp`.
2. Читай этот файл сверху вниз.
3. Не жди backend: используй демо-режим.
4. Если что-то сломалось, сначала проверь консоль браузера.
5. Если backend уже готов, сравни его JSON с разделом 3.
6. Если поля отличаются, исправь файл:

```text
frontend/app.js
```

7. Если нужно менять внешний вид, основные стили здесь:

```text
frontend/styles.css
```

8. После заметного изменения делай коммит:

```bash
git add frontend
git commit -m "Build frontend demo"
```

## 10. Как данные проходят через frontend

Главная схема:

```text
Пользователь вводит тикер
  -> app.js ловит отправку формы
  -> analyzeTicker() получает данные
  -> renderResult() выводит общий результат
  -> drawPriceChart() рисует график
  -> renderNews() выводит новости
```

Тебе важно запомнить: frontend не знает, откуда backend взял данные. Frontend получает готовый JSON и раскладывает его по странице.

### `index.html`

Это разметка страницы.

Здесь находятся элементы, которые потом заполняет JavaScript:

```html
<input id="ticker-input" />
<button id="submit-button">Анализ</button>

<strong id="stock-price"></strong>

<p id="ai-analysis"></p>
<div id="news-list"></div>
```

Если нужно добавить новый блок на страницу, сначала добавляй его сюда.

### `styles.css`

Это внешний вид страницы.

Здесь меняются:

- цвета;
- размеры;
- отступы;
- сетка колонок;
- карточки;
- внешний вид графика;
- адаптация под маленький экран.

### `mockData.js`

Это временная база данных для frontend.

Пока backend не подключен, данные берутся отсюда:

```javascript
const mockStocks = {
  AAPL: { ... },
  MSFT: { ... },
  TSLA: { ... }
};
```

Если хочешь добавить демо-тикер, добавь новый объект внутрь `mockStocks`.

### `app.js`

Это главный файл логики.

Здесь происходит:

- чтение тикера из поля ввода;
- проверка пустого значения;
- получение данных из mock или backend;
- обработка ошибок;
- вывод результата;
- отрисовка графика;
- вывод новостей.

## 12. Главные переменные в `app.js`

В начале файла есть переключатель:

```javascript
const USE_MOCK_API = true;
```

Если `true`, frontend берет данные из `mockData.js`.

Если `false`, frontend делает запрос в backend:

```javascript
const API_BASE_URL = "http://localhost:8000";
```

То есть при подключении backend нужно сделать так:

```javascript
const USE_MOCK_API = false;
const API_BASE_URL = "http://localhost:8000";
```

## 13. Главные функции в `app.js`

### `normalizeTicker(ticker)`

Принимает текст из поля ввода.

Делает:

- убирает пробелы по краям;
- переводит тикер в верхний регистр.

Пример:

```text
" aapl " -> "AAPL"
```

### `analyzeTicker(ticker)`

Это главная функция получения данных.

Принимает:

```javascript
ticker
```

Возвращает:

```javascript
data
```

Если включен демо-режим:

```javascript
return mockStocks[normalizedTicker];
```

Если подключен backend:

```javascript
fetch(`${API_BASE_URL}/api/analyze/${normalizedTicker}`)
```

То есть именно здесь frontend переключается между демо-данными и реальным сервером.

### `renderResult(data)`

Принимает готовый объект данных от `analyzeTicker()`.

Заполняет основные блоки страницы:

```javascript
resultTicker.textContent = data.ticker;
companyName.textContent = data.companyName;
stockPrice.textContent = formatPrice(data.price, data.currency);
companyDescription.textContent = data.companyDescription;
aiAnalysis.textContent = data.aiAnalysis;
```

Потом вызывает:

```javascript
drawPriceChart(data.priceHistory || [], data.currency);
renderNews(data.news || []);
```

### `drawPriceChart(history, currency)`

Принимает историю цены:

```javascript
[
  { date: "20.05", price: 184.1 },
  { date: "21.05", price: 185.4 }
]
```

И рисует график на:

```html
<canvas id="price-chart"></canvas>
```

Если Chart.js загрузился, график будет интерактивным:

- колесо мыши приближает и отдаляет график;
- перетаскивание двигает график по датам;
- кнопка `Сбросить масштаб` возвращает исходный вид;
- при наведении показывается tooltip с ценой.

Если данных меньше двух точек, график не ломается, а показывает сообщение, что истории цены недостаточно.

Если Chart.js не загрузился, сработает простая функция:

```javascript
drawFallbackPriceChart(history, currency)
```

Она рисует статичный график на canvas.

### `renderNews(newsItems)`

Принимает массив новостей:

```javascript
[
  {
    title: "Заголовок",
    publishedAt: "2026-05-29",
    summary: "Краткое содержание",
    url: "https://example.com",
    impact: "positive",
    impactReason: "Причина оценки"
  }
]
```

Создает карточку для каждой новости и добавляет ее в:

```html
<div id="news-list"></div>
```

### `setMessage(text, type)`

Показывает сообщение пользователю.

Например:

```javascript
setMessage("Получаем данные и готовим анализ...");
setMessage("Тикер не найден", "error");
```

### `setLoading(isLoading)`

Меняет состояние кнопки.

Когда идет запрос:

```text
Анализ -> Загрузка
```

Когда запрос закончился:

```text
Загрузка -> Анализ
```

## 14. Что backend должен вернуть

Frontend ждет от backend объект такого формата:

```json
{
  "ticker": "AAPL",
  "companyName": "Apple Inc.",
  "price": 190.12,
  "currency": "USD",
  "dailyChangePercent": 1.24,
  "priceHistory": [
    { "date": "20.05", "price": 184.1 },
    { "date": "21.05", "price": 185.4 }
  ],
  "companyDescription": "Описание компании.",
  "news": [
    {
      "title": "Заголовок новости",
      "publishedAt": "2026-05-29",
      "summary": "Краткое содержание новости.",
      "url": "https://example.com",
      "impact": "positive",
      "impactReason": "Почему новость считается положительной."
    }
  ],
  "aiAnalysis": "Итоговый аналитический текст."
}
```

Критически важные поля:

- `ticker`
- `companyName`
- `price`
- `currency`
- `dailyChangePercent`
- `priceHistory`
- `companyDescription`
- `news`
- `aiAnalysis`

Если backend отдаст `company_name` вместо `companyName`, frontend это поле не увидит.

## 16. Что делать, если после подключения что-то сломалось

### Ошибка `Failed to fetch`

Возможные причины:

- backend не запущен;
- backend запущен на другом порту;
- неправильный `API_BASE_URL`;
- backend не настроил CORS.

Что проверить:

```text
http://localhost:8000/api/analyze/AAPL
```

Если этот адрес не открывается, проблема на стороне backend или адреса.

### Пустое название компании

Скорее всего backend вернул другое имя поля.

Frontend ждет:

```json
"companyName": "Apple Inc."
```

А backend мог вернуть:

```json
"company_name": "Apple Inc."
```

Тогда нужно договориться: либо backend меняет поле, либо ты меняешь `app.js`.

### Нет графика

Frontend ждет:

```json
"priceHistory": [
  { "date": "20.05", "price": 184.1 }
]
```

Для нормального графика нужно минимум две точки.

Если backend вернул пустой массив, frontend покажет, что истории цены недостаточно.

### Нет новостей

Frontend ждет:

```json
"news": []
```

Если массив пустой, карточек новостей не будет. Это не ошибка frontend.

### Ошибка с CORS

Если в консоли браузера есть слово `CORS`, backend должен разрешить запросы с frontend-адреса.

Для разработки frontend обычно открыт здесь:

```text
http://localhost:5173
```

Backend должен разрешить этот origin.

## 17. Мини-чеклист перед сдачей

Проверь:

- `AAPL` показывает данные;
- `MSFT` показывает данные;
- `TSLA` показывает данные;
- неправильный тикер показывает ошибку;
- старая карточка исчезает после ошибки;
- график отображается;
- новости отображаются;
- AI-анализ отображается;
- frontend работает с `USE_MOCK_API = true`;
- frontend работает с `USE_MOCK_API = false`, когда backend запущен.
