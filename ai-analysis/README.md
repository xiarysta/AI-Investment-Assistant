# AI-анализ в MVP

Эта папка содержит только часть участника 3: AI-анализ, тексты, тестирование и сборка. Она не меняет `frontend/`, `backend/` и уже написанную документацию.

Пока backend и реальный AI API не готовы, здесь подготовлен локальный fallback в файле:

```text
ai-analysis/analysisRules.js
```

Он делает три вещи:

- определяет влияние новости как `positive`, `negative` или `neutral`;
- добавляет короткое объяснение влияния, если его нет в данных;
- формирует итоговый аналитический текст по цене, динамике и новостям.

## Как работает демо-анализ

Правила специально простые и прозрачные для защиты MVP:

- позитивные сигналы ищутся по словам вроде `рост`, `расширяет`, `стабильн`, `спрос`, `выручк`;
- негативные сигналы ищутся по словам вроде `снижа`, `риск`, `давлен`, `конкурен`, `затрат`, `марж`;
- если позитивных сигналов больше, новость считается положительной;
- если негативных сигналов больше, новость считается отрицательной;
- если сигналы равны или отсутствуют, новость считается нейтральной.

Итоговый текст включает:

- название компании и тикер;
- дневное изменение цены;
- динамику цены за доступный период;
- количество положительных, отрицательных и нейтральных новостей;
- позитивные факторы;
- риски;
- нейтральный контекст;
- дисклеймер, что это не инвестиционная рекомендация.

## Prompt для будущего AI API

Когда backend будет готов, можно использовать такой prompt как основу:

```text
Ты финансовый аналитический ассистент. Сформируй краткую аналитическую справку по акции.

Входные данные:
- тикер: {{ticker}}
- компания: {{companyName}}
- текущая цена: {{price}} {{currency}}
- дневное изменение: {{dailyChangePercent}}%
- описание компании: {{companyDescription}}
- новости: {{news}}

Требования к ответу:
1. Пиши на русском языке.
2. Объем: 100-200 слов.
3. Укажи общую оценку ситуации.
4. Отдельно назови ключевые позитивные факторы.
5. Отдельно назови риски.
6. Учитывай тональность новостей, но не выдумывай факты вне входных данных.
7. Не давай рекомендацию купить, продать или держать акцию.
8. В конце явно напиши, что это аналитическая справка, а не инвестиционная рекомендация.
```

## Что передать backend

Backend может вернуть уже готовые поля:

```json
{
  "news": [
    {
      "impact": "positive",
      "impactReason": "Причина оценки влияния"
    }
  ],
  "aiAnalysis": "Итоговый аналитический текст"
}
```

Если backend временно не вернет `impact`, `impactReason` или `aiAnalysis`, этот fallback можно подключить позже отдельным frontend-изменением.
